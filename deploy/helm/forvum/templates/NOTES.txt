PER-NAMESPACE MEMORY ISOLATION
------------------------------
This release runs an ISOLATED Forvum instance. Its memory — the SQLite ledger at
{{ .Values.forvumHome }}/state/forvum.sqlite —
lives on a PersistentVolumeClaim scoped to THIS namespace:

  {{- if .Values.persistence.enabled }}
  PVC: {{ include "forvum.pvcName" . }} ({{ .Values.persistence.size }}, {{ join "," .Values.persistence.accessModes }})
  {{- else }}
  WARNING: persistence is DISABLED — state lives in an emptyDir and is LOST on pod restart.
           Set persistence.enabled=true for a durable, isolated store.
  {{- end }}

Installing this chart into another namespace creates a separate Deployment + PVC + state database, so
namespace "{{ .Release.Namespace }}" cannot read another namespace's conversations or memory. Run one
release per team for per-team isolation.
